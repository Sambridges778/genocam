# 项目开发日志（GenoCam）

## 编写约定
- 提交信息遵循 Conventional Commits：
  - `feat`: 新功能
  - `fix`: 修复问题
  - `docs`: 文档变更
  - `chore`: 构建/工具/杂项
  - `refactor`: 代码重构（非功能）
  - `perf`: 性能优化
  - `test`: 测试相关
  - `build` / `ci`: 构建流水线与持续集成
- 版本号使用语义化：`vX.Y.Z`。
- 主分支：`main`；建议后续使用 `feat/*`、`fix/*` 分支工作流。

## 工作记录模板
```
日期：YYYY-MM-DD
类型：feat/fix/docs/chore/refactor/perf/test
变更：简要描述改动与目的
影响：涉及模块/文件与可能影响的功能
备注：链接需求/任务编号或参考文档
```

## 里程碑追踪
- M1：相机预览/拍照与基础胶片预设（LUT+曲线）。
- M2：底片与重冲洗、曝光补偿、色温/色调、构图网格。
- M3：录像与复古视频模式（8mm/16mm/VHS/DCR/电影机），基础防抖。
- M4：双重曝光、镜头附件（鱼眼/星光/色散/ND）、滤色片、导入现有照片。
- M5：性能调优、资源优化、国际化与可访问性、测试矩阵与发布。
