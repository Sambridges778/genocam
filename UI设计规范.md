# GenoFood — HarmonyOS UI 设计规范补充（iOS风格，对标 Too Good To Go）

目的
- 明确 GenoFood 在鸿蒙 ArkUI 上的 UI 设计理念与页面蓝图，整体风格参考 iOS，
  体验全面对标 Too Good To Go（TGTG）。
- 统一单位 vp/fp；遵循现有规范：最小触达 48vp、4/8 栅格、圆角 12/16vp、
  屏内 SelectableText.rich 红色错误提示。

一、设计理念（Design Principles）
- Soft Minimalism：柔和极简，强调内容第一、卡片/列表为主、低噪点界面。
- Sustainable Tone：传达环保与节约理念，视觉简洁不喧宾夺主。
- iOS 风格节奏：顶部导航与分栏、底部 Tab Bar、细腻动效（轻微缩放/淡入）。
- 可预期与可信：下单—凭证—到店核销路径清晰，状态与反馈明确。
- 一致性与可达性：文案、间距、控件尺寸保持一致；无障碍色彩对比满足要求。

二、信息架构与导航（对标 TGTG）
- 顶层导航：Bottom Tab（5 项建议）
  1) 首页（列表）  2) 地图  3) 订单  4) 收藏  5) 我的
- 二级导航：顶部导航栏（返回/标题/操作）、筛选抽屉/弹窗。
- 关键路径：发现→选择时间段→下单→生成凭证→到店核销→评价与收藏。

三、核心页面蓝图（ArkUI 结构参考）
- OnboardingPage：品牌价值说明、权限引导（位置/通知）。
- HomeListPage：附近门店列表，支持筛选（距离/价格/品类/时间段）。
- HomeMapPage：地图查看门店位置与可订状态。
- ShopDetailPage：门店信息、礼包说明、时间段选择、剩余数量与价格说明。
- CheckoutPage：订单确认与付款占位（支付方式后续指定）。
- OrderSuccessPage：凭证展示（二维码/数字码），支持提醒与路线入口。
- OrdersPage：订单列表（待取餐/已完成/已取消）。
- OrderDetailPage：订单详情、凭证、时间段、核销状态。
- FavoritesPage：收藏门店列表。
- ProfilePage：个人信息、设置、通知开关、隐私说明与协议。

四、版式与尺寸（统一 vp/fp）
- 栅格：4/8 间距系统；页面边距建议 16vp；卡片内间距 12–16vp。
- 角与边：卡片圆角 12/16vp；弹窗与浮层圆角 16vp。
- 导航与工具条：
  - 顶部栏高度 ~56vp；标题使用中等字重；右侧为操作（筛选/分享）。
  - 底部 Tab Bar 高度 ~64vp；图标 24vp；标签采用短文案。
- 列表卡片：门店卡片高度随内容自适应，图文信息层次不超过 3 层。
- 交互尺寸：最小触达 48vp；关键按钮（如“预约”）≥56×48vp。

五、组件规范（对标 TGTG 的卡片/凭证/过滤）
- 门店卡片（StoreCard）
  - 结构：缩略图（图片占位：黄色）、店名/评分（文本）、品类标签、
    价格信息、可预约时间段与剩余数量、收藏按钮。
  - 交互：整卡可点进详情；右下角为主要操作（“预约”）。
- 筛选与排序（FilterSheet）
  - 条件：距离、价格区间、品类、取餐时间段；支持重置与应用。
- 凭证卡（Voucher）
  - 内容：二维码（或数字码）、门店地址与时间段、到店指引。
  - 行为：亮度提示（便于扫码）、到店提醒开关、复制码。
- 订单卡（OrderCard）：状态图标、取餐时间窗、门店信息与按钮（查看/评价）。
  
六、视觉与动效（iOS 风格）
- 色彩：以中性背景与轻微分层体现层次；强调色使用统一 `accent` 令牌。
- 字体：标题/副标题/正文按比例（参考 fp 标尺），正文行高稳定可读。
- 动效：按钮按压轻微缩放，页面切换淡入/滑入；动画时间保持轻量与一致。
- 空态与错误：统一卡片式空态；错误使用屏内 SelectableText.rich 红色文案。

七、无障碍与国际化
- 触达与对比度：关键按钮与文本满足可达性与对比度要求。
- 文案长度：尽量短句；多语言预留空间（中文/英文等）。
- 符号与图标：提供可读性强的图标与文本备选说明。

八、Figma 设计要求（iOS 风格画板与命名）
- 文件结构（Pages）：01_Onboarding、02_Home_List、03_Home_Map、
  04_Shop_Detail、05_Checkout、06_Order_Success、07_Orders、
  08_Order_Detail、09_Favorites、10_Profile、11_Components。
- 画板与命名：
  - iPhone 画板（iOS 风格），命名规范：`PageName / Variant`。
  - 组件命名：`Comp/StoreCard`、`Comp/Voucher`、`Comp/FilterSheet`、
    `Nav/TopBar`、`Nav/TabBar`、`List/Cell`。
- Auto Layout：卡片/列表统一使用自动布局，适配文案长度与动态内容。
- 令牌与样式：Color/Type/Radius/Shadow 使用 Style 命名（不在此处定义具体值）。
- 媒体占位约定（遵循项目规则）：
  - 图标用红色方块占位；图片用黄色；视频用蓝色；其他文件用绿色。

九、ArkUI 参考（结构与命名）
- 组件：StoreCard、FilterSheet、Voucher、OrderCard、TopBar、TabBar。
- 命名与目录：pages 与 components 分层，逻辑尽量使用无状态组件与组合。
- 性能：列表分页/懒加载；const 小部件优先；避免不必要重绘。

十、验收与度量（不设具体目标值）
- 路径完成率：发现→下单→凭证→核销闭环无阻碍。
- 首屏与滚动：首页列表/地图滑动流畅；卡片点击反馈一致。
- 无障碍检查：触达、对比度、字号放大后布局稳定。

附录：文案与图标风格
- 文案主张节约与环保，简洁温暖；避免冗长描述。
- 图标线性、简洁；状态色不刺眼，优先以层次与对比传达重点。
